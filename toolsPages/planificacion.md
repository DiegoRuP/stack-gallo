# Planificación y Gestión de Proyectos

Antes de comenzar a escribir código o crear documentos, es importante planificar y aclarar ideas no solo en la mente, si no también de forma digital.

Estas herramientas nos ayudan a organizar desde apuntes de clase hasta proyectos complejos en equipo, asegurando que todos estemos en la misma página.

---

## Gestión de tareas y apuntes

Esta categoría se centra en capturar y organizar información. Son nuestros "cuadernos digitales" y gestores de tareas del día a día.

### Notion
En mi opinión, es una de las herramientas "todo en uno" más potentes. Nos sirve para tomar apuntes de clase, crear wikis personales o para el equipo, y gestionar tareas en tableros Kanban.

* **Cómo nos sirve:** Podemos usarla para llevar el control de todas nuestras materias, enlazar notas, crear bases de datos de tareas y hasta planificar proyectos enteros.
* **Beneficio:** Tiene un plan "Plus" (antes Pro) gratuito para estudiantes con solo registrar nuestro correo institucional.
* **Enlace:** [https://www.notion.so](https://www.notion.so)

### Obsidian
Esta es una herramienta para quienes se toman muy en serio sus notas. Funciona con archivos Markdown locales en nuestra computadora, lo que la hace muy rápida y privada.

* **Cómo nos sirve:** Es ideal para construir una "base de conocimiento" o un "segundo cerebro". Su fortaleza es la capacidad de enlazar ideas y verlas en un gráfico. Perfecta para conectar conceptos de diferentes materias o para una tesis.
* **Beneficio:** Es 100% gratuita para uso personal.
* **Enlace:** [https://obsidian.md](https://obsidian.md)

### Trello
Es la simplicidad hecha tablero Kanban. Es increíblemente visual y fácil de usar.

* **Cómo nos sirve:** Es la mejor opción para proyectos en equipo donde no necesitamos la complejidad de Notion. Podemos crear tarjetas para tareas y moverlas por columnas como "Pendiente", "En Proceso" y "Terminado".
* **Beneficio:** Su plan gratuito es más que suficiente para cualquier proyecto universitario.
* **Enlace:** [https://trello.com](https://trello.com)

### Evernote
Es uno de los clásicos para tomar notas. Su gran fortaleza siempre ha sido la sincronización y su "Web Clipper" para guardar artículos o páginas web.

* **Cómo nos sirve:** Es útil si necesitamos un lugar rápido para guardar todo tipo de contenido (texto, PDFs, imágenes, capturas web) y tenerlo disponible en todos nuestros dispositivos al instante.
* **Beneficio:** Tiene un plan gratuito, aunque es más limitado que antes.
* **Enlace:** [https://evernote.com](https://evernote.com)

## Diagramas y flujos

Estas herramientas son esenciales para el pensamiento visual. Nos permiten crear desde diagramas de flujo hasta arquitecturas de software complejas.

### Lucidchart
Es una de las herramientas web más profesionales para diagramación. Es colaborativa en tiempo real.

* **Cómo nos sirve:** Perfecta para trabajos formales de clase, como diagramas UML, diagramas de Entidad-Relación (bases de datos) o flujos de procesos.
* **Beneficio:** Ofrece un plan educativo gratuito que nos da acceso a funciones premium.
* **Enlace:** [https://www.lucidchart.com](https://www.lucidchart.com)

### Draw.io (diagrams.net)
Esta es la alternativa gratuita y de código abierto a Lucidchart. Es sorprendentemente poderosa.

* **Cómo nos sirve:** Hace prácticamente todo lo que hace Lucid, pero gratis. Podemos usarla en la web sin registrarnos o incluso como una extensión de VS Code para guardar los diagramas junto a nuestro código.
* **Beneficio:** Es totalmente gratuita.
* **Enlace:** [https://app.diagrams.net](https://app.diagrams.net)

### Excalidraw
Es mi favorita para lluvias de ideas rápidas. Es una pizarra virtual que le da a todo un aspecto de "dibujado a mano".

* **Cómo nos sirve:** Es ideal para esbozar una arquitectura de sistema, un *wireframe* de una app o explicar un concepto a un compañero. Es menos formal pero muy rápida y colaborativa.
* **Beneficio:** Es gratuita y de código abierto.
* **Enlace:** [https://excalidraw.com](https://excalidraw.com)

### Mermaid
Esta es diferente. Es una herramienta que nos permite crear diagramas *escribiendo código* (o, mejor dicho, una sintaxis similar a Markdown).

* **Cómo nos sirve:** Es increíble para la documentación. Podemos escribir un diagrama de flujo o de clases directamente en nuestros archivos `README.md` en GitHub y se mostrará visualmente. Nos permite versionar nuestros diagramas junto con el código.
* **Beneficio:** Es un estándar de facto en la documentación técnica.
* **Enlace:** [https://mermaid.js.org](https://mermaid.js.org)

### Graphviz
Es una herramienta clásica y muy potente para generar gráficos complejos a partir de código (usando un lenguaje llamado DOT).

* **Cómo nos sirve:** Mientras que Mermaid es buena para flujos, Graphviz es excelente para visualizar estructuras de datos complejas, redes o jerarquías de clases. Muchas otras herramientas la usan por debajo.
* **Beneficio:** Es gratuita y extremadamente potente para gráficos generados automáticamente.
* **Enlace:** [https://graphviz.org](https://graphviz.org)
