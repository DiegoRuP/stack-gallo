# Documentación y Colaboración

Un proyecto no solo es código, también es comunicación. Escribir una buena documentación y tener canales de equipo claros es lo que diferencia un proyecto de clase de un proyecto profesional.

Estas herramientas nos ayudan a explicar qué hace nuestro código y a mantenernos sincronizados como equipo.

---

## Documentación

La documentación es la guía de usuario de nuestro proyecto. Va desde el `README.md` hasta manuales técnicos complejos.

### Markdown
Es un lenguaje de marcado ligero que usamos para dar formato a texto plano. Es el lenguaje con el que está escrito este sitio web.

* **Cómo nos sirve:** Es fundamental. Lo usamos para escribir `README.md` claros en GitHub, para tomar notas (en Obsidian o Notion) y para escribir la documentación de nuestros proyectos. Es increíblemente fácil de aprender.
* **Beneficio:** Es gratuito, universal y está integrado en casi todas las plataformas de desarrollo.
* **Enlace:** [https://www.markdownguide.org/](https://www.markdownguide.org/)

### LaTeX
Esto es mucho más que Markdown. Es un sistema de composición de textos de alta calidad. Si Markdown es para notas rápidas, LaTeX es para documentos formales.

* **Cómo nos sirve:** Es el estándar de oro en el mundo académico y científico, especialmente si necesitamos escribir fórmulas matemáticas complejas. Lo usaremos para artículos de investigación, reportes de laboratorio y, sobre todo, para la tesis.
* **Beneficio:** Es gratuito y produce los documentos PDF más profesionales que existen. (Plataformas como [Overleaf](https://www.overleaf.com/) nos permiten usarlo en la nube).
* **Enlace:** [https://www.latex-project.org/](https://www.latex-project.org/)

## Comunicación de equipo

Estas son las plataformas que usamos para la comunicación del día a día, hacer preguntas rápidas y organizar el trabajo en equipo.

### Slack
Es la herramienta de comunicación estándar en la mayoría de las empresas de tecnología. Funciona a través de "canales" (para temas específicos) e hilos de conversación.

* **Cómo nos sirve:** Es excelente para proyectos de equipo formales. Podemos crear canales como `#frontend`, `#backend` o `#dudas` para mantener las conversaciones ordenadas y evitar saturar un solo chat.
* **Beneficio:** Tiene un plan gratuito muy generoso que es más que suficiente para un equipo universitario.
* **Enlace:** [https://slack.com/](https://slack.com/)

### Discord
Empezó como una herramienta para *gamers*, pero ahora es usada por miles de comunidades de código abierto y clubes estudiantiles.

* **Cómo nos sirve:** Su gran fortaleza son los canales de voz. Podemos crear una sala de voz donde los miembros del equipo pueden "entrar y salir" para trabajar juntos, como si estuvieran en la misma habitación. Es más informal y fluido que Slack.
* **Beneficio:** Es completamente gratuito y excelente para construir comunidades.
* **Enlace:** [https://discord.com/](https://discord.com/)
